Create Migration Project

A project is the business wrapper for a migration initiative. Create it under the right organization and business unit first; attach source and target database connections separately as the work moves from assessment to conversion, load, CDC, and validation.

Assessment Scope

Source First

Target Strategy

Optional For Assessment
  • Target PostgreSQL connection is not needed to run the first Oracle assessment
  • Choose target platform after sizing, compatibility scoring, extension review, and HA/DR requirements
  • Attach one or more target database connections later for conversion, deployment, load, CDC, and validation

Governance

Enterprise Guardrail
  • Assign project owner, reviewer, and approver roles before conversion or cutover
  • Keep source and target credentials in customer-managed vaults or agent-local secret stores
  • Record assessment evidence and approvals under the project audit trail

Create Database Connection

Database connections are reusable endpoint profiles under a business unit or project. Synqora Cloud stores metadata and validation evidence; the customer-side agent performs the real network and credential checks.

Endpoint Metadata

No Password Stored

Validation Plan

Agent Executed
  • Check network reachability from the selected Synqora agent
  • Verify least-privilege permissions for discovery, conversion, load, CDC, or validation role
  • Store validation evidence and failures under the project audit trail
